活動頁版型開發

活動網頁設計 Web Design Nov. 2020 - Mar. 2021 Designer、F2E

PROJECT

這是一個電商商品版型的demo，開發了數十種版型，只需一鍵複製，貼到準備好的文件中，就能快速產生一個有商品的頁面，設計給內部同仁使用。

WHY？

一開始只是為了把公司舊有的檔案整理一番，因為長期沒有人維護，程式碼沒有妥善命名，被貼來貼去，髒到每次要使用都需要花一段時間整理，或是常常會破版。後來決定自己重寫，但這問題還是沒有完全解決，當需要拿別人的檔案來用時，還是會遇到許多麻煩，於是利用上班零碎的時間，將自己改寫的版型整理成的一個demo頁，供同事使用。

解決什麼問題？

  1. 版型名字都一樣，貼來貼去就壞了，也很難分辨。
  2. 沒有明確的架構，修改與調整不易。
  3. 舊有的float:left排版易跑位。
  4. 文字大大小小、顏色不夠統一。
  5. 要簡單易懂、不會程式也能使用。

構思架構

訪問了幾個PM，問問他們產品可能會放的內容及需求，針對這些固定的內容，設計一個可以共用的架構(HTML)，再用CSS去做變化。主要有圖片與文字，架構如下： 


    <li>
        <img src="圖片"/>
        <span class="font">
            <h4>標題</h4> 
            <p>內文</p>
            <span class="price">價錢</span>
        </span>
    </li>

我把文字與圖片分開，這樣方便做不同的排版。

樣式修改

接下來是一個大工程，花費了我許多時間在寫每個版型的樣式，尤其在詞窮不知道怎麼命名的時候，還有特殊版型變化的時候，以及調整手機版的時候…，在CSS針對每個版型都有不同的調整，部分數值會經常使用、修改的，用：root管理。也為整個網頁定義固定的文字大小、字型、間距、顏色，讓整體更一致。
捨棄float排版，改用flex，解決常常跑位、破版、內容不一無法對齊的狀況，排版與變化也更容易些。 在行動裝置上的版型更為多元，也測試了字型大小在各裝置上是否易讀，操作是否順暢。

最後成果

我將寫好的版型，依照一排商品的數量分類整理在一個demo頁面，附上規格書。想著如果可以讓不會寫程式的PM，也能自己產出頁面，那應該會省很多事，於是多了一個複製程式碼的功能，只要將它貼在我準備好的文件內即可。

最後在三月初釋出給同事們使用，後來有針對一些問題修正，也陸續新增一些功能與版型，一直使用到現在。使用此公版減少了設計同仁50%以上的切版時間，有更多心力與時間著墨在視覺及版型的變化上。有此demo作為參考依據，與企劃、PM溝通能更有效率。
